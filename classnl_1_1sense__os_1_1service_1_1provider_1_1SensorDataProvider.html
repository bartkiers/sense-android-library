Detailed Description

ContentProvider that encapsulates recent sensor data. The data is stored in the devices RAM memory, so this implementation is more energy efficient than storing everything in flash. This does mean that parsing the selection queries is quite a challenge. Only a very limited set of queries will work:

  • sensor_name = 'foo'
  • sensor_name != 'foo'
  • timestamp = foo
  • timestamp != foo
  • timestamp > foo
  • timestamp >= foo
  • timestamp < foo
  • timestamp <= foo
  • combinations of a sensor_name and a timestamp selection
